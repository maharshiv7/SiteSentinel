Threat Dictionary

Understand the critical vulnerabilities and missing headers detected by our security engine.

Clickjacking

Missing: X-Frame-Options

An attacker uses transparent or opaque layers to trick a user into clicking on a button or link on another page when they were intending to click on the top-level page.

The Fix: Configure your server to send the X-Frame-Options: DENY header to prevent your site from being embedded in malicious iframes.

Cross-Site Scripting (XSS)

Missing: Content-Security-Policy

XSS allows attackers to inject malicious scripts into the web pages viewed by other users. This can lead to stolen session cookies, hijacked accounts, and defaced websites.

The Fix: Implement a robust Content-Security-Policy (CSP) header that restricts where scripts can be loaded and executed from.

Protocol Downgrade (MITM)

Missing: Strict-Transport-Security

A Man-In-The-Middle (MITM) attacker forces the user's browser to connect over insecure HTTP instead of secure HTTPS, allowing them to intercept passwords and sensitive data.

The Fix: Enable Strict-Transport-Security (HSTS) to force browsers to only connect via HTTPS, even if the user types 'http://'.
